# DESIGN.md — VostokLike ゲーム設計書

## 1. ゲームコンセプト・方向性

### どんなゲームを目指しているか

- サバイバル / Extraction / 建築 / 文明復興 / 観測シミュレーションを融合したゲーム
- 「探索の緊張感」と「帰還時の解放感」をコア体験にする
- プレイヤーは世界の主人公ではなく、「世界を観測し関与する存在」
- 世界そのものが自律的に変化・発展することを重視
- 将来的には文明再建から宇宙進出・未知文明接触まで視野

### 他ゲームとの差別化ポイント

- 「帰還理由」と「遠征理由」を両立させる
- 感情コストを利用したシステム設計（Vostokの猫システム的思想）
- NPCが建築物を利用することで建築が"完成"する
- プレイスタイルごとに異なる帰還理由を持てる
- 世界がプレイヤー不在でも進行する
- 文明・文化・思想レベルまで発展可能な世界シミュレーション
- 「効率」より「感情」を重視した設計思想

### 感情・体験として重視していること

- 無事に帰還できた安心感
- 長距離遠征の不安
- 拠点への愛着
- 文明を取り戻していく感覚
- NPCが生活する世界への没入感
- 自分の建築が社会の一部になる感覚
- 生存だけでなく「存在している実感」
- 時間経過による喪失感・文明変化
- プレイヤーの選択によって世界が変わる感覚

## 2. コアループ

### 基本サイクル

1. 拠点準備
2. 遠征
3. 探索 / Loot / 戦闘 / 発見
4. 生還
5. 拠点帰還
6. 整理 / 建築 / NPC交流 / 発展
7. 次の遠征準備

### 長期ループ

```
小規模拠点 → 集落化 → 文明復興 → 国家レベル発展
→ 宇宙進出 → 未知文明観測 / 接触 → 文明崩壊後の再復興
```

---

## 3. 各システムの設計意図

### 感情アンカー

プレイヤー自身が「帰りたい」と思える理由を作る。効率ではなく感情で選択を悩ませる。

具体例：猫 / 音楽 / コレクション / NPC / 農業 / 発電機 / 自室

### NPCシステム

世界に生活感を与え、「世界は誰かの仕事で出来ている」を表現する。文明復興感を強化。

要素：前職 / 才能 / 思想 / 性格（MBTI的特性）/ 派閥適性

### 建築システム

「見た目」ではなく「利用されて完成する建築」。NPCが利用することで社会性が生まれ、建築勢にも強い動機を与える。

### デス / 救助システム

キャラクターへの愛着・命の重さ・ドラマ生成を目的とする。「プレイヤー≠キャラクター」構造を活かす。

要素：捕獲 / 救出ミッション / 別キャラ操作 / GANTZ風蘇生

### 文明シミュレーション

プレイヤーが観測者になり、世界が自律して進行する感覚を作る。時間経過による文明変化と長期的ロマンを演出。

## 4. 将来実装したいもの

### 優先度：高

- FPS探索・Loot
- 建築・拠点
- NPC利用建築
- 感情アンカー
- 遠征 / 帰還ループ

### 優先度：中

- NPC個性・派閥
- 集落発展
- 車両・電力・農業
- 音楽システム

### 優先度：低（超長期）

- 宇宙探索・光速超越
- 文明観測・未知文明接触
- 文明崩壊シミュレーション
- 国家形成・AI生成歴史

## 6. 決定した制約・ルール

- ファストトラベルは現時点で非採用寄り
- 「効率最優先ゲーム」にしすぎない
- プレイヤー中心世界にしすぎない
- 世界が自律して動くことを重視
- 建築はNPC利用前提で考える
- 感情設計を最優先にする
- 個人開発で実現可能な範囲を意識
